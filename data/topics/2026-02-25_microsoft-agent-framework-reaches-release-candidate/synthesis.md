## TL;DR

Microsoft has released Release Candidate versions of its unified Agent Framework for both .NET and Python, consolidating what were previously two distinct pathways—AutoGen and Semantic Kernel—into a single open-source SDK. The framework delivers multi-agent orchestration capabilities, function tooling, and native support for the A2A, AG-UI, and MCP protocols, with General Availability targeted for Q1 2026.

---

### Key Developments

**1. Unification of Previously Separate Ecosystems**
The RC marks the formal convergence of AutoGen and Semantic Kernel into one cohesive SDK. For developers, this eliminates the need to choose between ecosystems or maintain separate codebases. The unified framework preserves the strengths of both: AutoGen's multi-agent conversation patterns and Semantic Kernel's enterprise-grade orchestration and integration capabilities.

**2. Protocol Support and Interoperability**
Microsoft's Agent Framework now natively supports three key protocols: **A2A** (Agent-to-Agent communication), **AG-UI** (agent user interface primitives), and **MCP** (Model Context Protocol). This positions the framework as a potential interoperability layer across the broader agentic AI landscape, allowing agents built on this SDK to communicate with external agents and tools that adhere to these emerging standards.

**3. Cross-Language Parity (.NET and Python)**
The simultaneous RC release for both .NET and Python reflects Microsoft's commitment to serving diverse developer ecosystems. Both implementations share feature parity, enabling teams to collaborate across language boundaries without sacrificing capabilities—a critical factor for enterprise adoption.

---

### Why This Matters

For organizations building agentic AI systems, this consolidation provides a canonical, future-proof path forward while reducing integration complexity and long-term maintenance burden.
